# 1. Introduction

## a. Purpose of the System

- The system aims to establish a peer-to-peer learning platform for students.
- It enables learners to clear academic queries through mutual cooperation.
- The platform facilitates connections between students seeking help and those possessing relevant subject expertise via chat-based interaction.
- It also promotes an engaging, discussion-oriented learning environment.

## b. Scope of the Project

- The project involves the creation of an online peer learning web application.
- Students are provided with the facility to post academic questions.
- The system intelligently links students with appropriate peers based on relevance.
- Discussions continue until the raised doubts are satisfactorily resolved.
- The platform is designed to be scalable, allowing future enhancements and additional features.

## c. Definitions, Acronyms

- **Peer Learning** – A learning approach based on interaction among students.
- **User** – A student who is registered on the platform.
- **Admin** – An authority responsible for supervising and managing the system.
- **Doubt** – An academic query or problem submitted by a student.

# 2. Overall Description

## a. Product Perspective

- The application functions as a standalone, web-based system.
- It can be accessed using common web browsers.
- The system operates independently without reliance on existing institutional academic platforms.
- Its architecture supports adaptability for future integrations.

## b. User Classes

- **Student**
  - Submits academic queries
  - Shares knowledge in specific subjects
  - Actively participates in peer discussions

- **Administrator**
  - Oversees user accounts
  - Tracks and supervises system activities
  - Ensures smooth operation and maintenance of the platform

## c. Operating Environment

The Peer Connect system will function under the following conditions:

### Web Application

- A browser-based platform intended for students within a particular academic campus.
- Provides a collaborative space for academic discussions and doubt resolution.

### Supported Browsers

- Google Chrome
- Mozilla Firefox

### Device compatibility

- Optimized responsive layout for desktop systems.

### Server Environment

- Backend Framework: Node.js / Django / Flask
- Database: MongoDB / MySQL

## d. Assumptions and Dependencies

- Users are expected to have a basic understanding of web-based applications.
- The academic information entered by users is presumed to be accurate.
- A stable internet connection is necessary for system access.
- System functionalities may evolve with updates and improvements over time.
